# Education Leader Portraits — story drafts

The Education Leaders Portraits series carries short-form leader profiles ShikshaLokam publishes — companion to the longer blogs. One `### Portrait — <Leader Name>` section per leader. Status moves `wip → review-ready → locked`. Pairs with Thursday's portrait slot on the [[captions]] page.

## How this page works

- Sonal in chat: *"new portrait — [Leader Name] at [Location]"* → Claude adds the section with the portrait template, `status: wip`, and pulls any relevant context from `wiki/sources/` (e.g. partner-state assets for that leader's state).
- Sonal pushes detail across sessions → body fills in, attribution + consent flag set, `status: review-ready`.
- Once Sonal copies the title back as confirmation → `status: locked`, optional Thursday caption variant lands in [[captions]] page.

---

## Portrait template

Each portrait carries these fields. Keep them in this order so the HTML mirror can lay them out consistently.

```
### Portrait — <Leader Name>
- **Status:** wip | review-ready | locked
- **Role:** <Title at Organisation>
- **Location:** <Village / Town / District, State>
- **Lead image:** <URL or Drive ref, placeholder OK>
- **Hook (1 line):** <the one sentence that opens the portrait>
- **Body (≤250 words):** <prose>
- **Forward-looking close (1 line, ideally in the leader's own words):** <quote or paraphrase>
- **Attribution / consent:** <named consent status: verbal | written | pending>
- **Source thread:** <chat session date / Drive doc / partner-state asset folder>
```
