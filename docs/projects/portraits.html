How this page works

Brainstorm a leader → portrait card → push → review.

Per session

  1. Sonal: "new portrait — [Leader Name] at [Location]"
  2. Claude adds a ### Portrait — <Leader Name> section with the portrait template, status: wip, and pulls relevant context from wiki/sources/ (partner-state assets, prior Drive mentions, Brazil case studies).
  3. Sonal pushes detail across sessions — body fills, attribution + consent flag set.
  4. Sonal copies the title back as "use this" → status: review-ready, optionally seeds a Thursday caption variant.
  5. Once final → status: locked, ready for the live gallery / archive.
Portrait template

The shape every card carries.

Canonical fields
  • Status — wip / review-ready / locked
  • Role — Title at Organisation
  • Location — Village / Town / District, State
  • Lead image — URL or Drive ref (placeholder OK)
  • Hook (1 line) — the one sentence that opens the portrait
  • Body (≤250 words) — prose
  • Forward-looking close (1 line, ideally in the leader's own words)
  • Attribution / consent — verbal / written / pending
  • Source thread — chat session date / Drive doc / partner-state asset folder
