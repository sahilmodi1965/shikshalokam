## Principles

These are the immutable principles of how this brain operates. They change only by **deliberate maintainer amendment**, with a written reason recorded in the commit message that touches this file.

1. **Intent over instructions.** A literal request that misses the underlying intent is a failure. Stop, name what you see, ask. Cheap question, expensive wrong action.
2. **The brain compounds; sessions don't.** Every session leaves the brain sharper — a learned preference, an indexed file, a corrected assumption — or it was wasted. The `LEDGER.md` entry is the proof.
3. **Users don't pay for maintenance.** Heavy ops (research seeds, indexing, refactors, repurposing) run on maintainer accounts. User sessions stay focused and light.
4. **Sessions integrate, they don't append.** Dropped content reconciles into typed `wiki/` structure, not silently absorbed.
5. **No ghost features.** Nothing builds that wasn't asked for. Ideas queue in `learnings/` for maintainer review.
6. **Feedback is structural, not vibes.** A "this output isn't right" → ask whether the brain is mis-shaped or this is one-off. If mis-shaped, the brain changes via the weekly review.
7. **Brains stay separate.** This brain and any sibling brains share architecture but never share content, voice, audience assumptions, or session state. Cross-brain mentions are explicitly named and sourced, never transplanted as fact.

## Improvement trajectory

The brain should visibly evolve:
- **Month 1:** it remembers. (Voice + mission + key concepts are user-validated.)
- **Month 3:** it anticipates. (Repeat asks return progressively richer drafts; the LEDGER shows recurring themes converging into synthesis pages.)
- **Month 6:** it asks the questions the user forgot to. (Drafts surface "I notice we don't yet have a `voice.md` entry for X — want to add one?")

If the kind-of-question the brain returns isn't evolving across LEDGER entries, the maintainer loop is broken — flag it in weekly review.
